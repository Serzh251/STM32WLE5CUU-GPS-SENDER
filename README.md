# 🛰️ GPS LoRa Tracker

Автономный маяк для удалённой передачи геолокации и состояния аккумулятора по радиоканалу LoRa. Оптимизирован для минимального энергопотребления за счёт циклического режима работы и глубокого сна.

## ⚙️ Принцип работы

1. **Пробуждение** по аппаратному таймеру.
2. **Считывание координат** с GPS-модуля (ожидание стабильного 3D-фикса).
3. **Замер напряжения АКБ** через встроенный АЦП и делитель напряжения.
4. **Формирование пакета**: `[Широта | Долгота | Напряжение | CRC]`.
5. **Отправка пакета** через LoRa-модуль на базовую станцию.
6. **Ожидание ACK** (подтверждения приёма) в течение заданного таймаута.
7. **Ретрай**: если ACK не получен → повтор отправки (макс. **3 попытки**).
8. **Переход в сон**: микроконтроллер и периферия переводятся в режим глубокого сна до следующего цикла.

## 🛠️ Требуемое оборудование

| Компонент | Рекомендуемый вариант                   | Примечание                                      |
| :--- |:----------------------------------------|:------------------------------------------------|
| **Микроконтроллер** | STM32WLE5CCU6 модуль E77-900M22S        |                                                 |
| **GPS-приёмник** | `u-blox MAX-M10`                        | NMEA + резервное питание для быстрого фикса     |
| **LoRa-модуль** | встроен в микроконтроллер               | Настройка частоты и мощности зависит от региона |
| **Датчик напряжения** | Делитель + АЦП МК                       | Калибровка обязательна                          |
| **Питание** | `Li-ion 18650`        | Рекомендуется ёмкость ≥ 2000 мА·ч               |

## 📝 Конфигурация (`config.h`)

```c
// LoRa параметры
#define LORA_FREQ       433.0   // МГц (433 / 868 / 915)
#define LORA_BW         125.0   // кГц
#define LORA_SF         9       // Spreading Factor (7-12)
#define LORA_TX_POWER   17      // дБм

// Сетевые параметры
#define DEVICE_ID       0x01    // Уникальный ID устройства
#define ACK_TIMEOUT_MS  2000    // Таймаут ожидания подтверждения (мс)
#define MAX_RETRIES     3       // Макс. количество попыток отправки

// Энергосбережение
#define SLEEP_INTERVAL_S 300    // Интервал пробуждения (сек)
#define VBAT_LOW_V      3.0     // Порог низкого заряда (В)
```

| Пин | Режим (`pinMode`) | Назначение | Примечание                                     |
| :--- | :--- | :--- |:-----------------------------------------------|
| `PA2` | `INPUT` / `Serial RX` | Приём NMEA от GPS | Чтение данных (`Serial.available()`)           |
| `PA3` | `OUTPUT` / `Serial TX` | Отправка команд в GPS | Конфигурация/управление модулем                |
| `PA6` | `OUTPUT` | Разрешение передачи LoRa (`TX_EN`) | `digitalWrite(PA6, HIGH)` перед `radio.send()` |
| `PA7` | `OUTPUT` | Разрешение приёма LoRa (`RX_EN`) | `digitalWrite(PA7, HIGH)` при ожидании ACK     |
| `PA9` | `OUTPUT` | Управление питанием GPS | `HIGH` = вкл, `LOW` = выкл (экономия)          |
| `PA11`| `INPUT` | Аналоговый вход АЦП (АКБ) | `analogRead(PA11)` → пересчёт в Вольты         |
| `PA12`| `OUTPUT` | Ключ цепи замера АКБ | `HIGH` = включить делитель, замер, затем `LOW` |
